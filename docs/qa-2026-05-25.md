## Q1. 既存の FRED や Our World in Data との違いは?

A. FRED は米国マクロ経済データ、Our World in Data はグローバル社会課題に特化していますが、EIC Data は **日本のエネルギー × 金融の交差点** にフォーカスしています。日本独自の電力卸価格 (JEPX 10 エリア) + 気象データ (JMA 9 地点) + 燃料 + 金融 (USD/JPY + JGB + 米マクロ) を**横断的に Insight 化**し、一次出典 URL + as-of 日付 + 引用フォーマット 3 形式 (BibTeX/Chicago 17/APA 7) を全系列に付与する点が独自です。研究者は引用元として、ジャーナリストは数字の根拠として、実務者は意思決定の前提として「2 クリックで一次出典に辿れる」基準点として使えます。

## Q2. データの出典は信頼できるのか?

A. 全 105 系列が **政府機関または公的指定機関の一次データ**です: JEPX (電力卸価格)、JMA (気象庁)、METI (経済産業省 電力調査統計)、BOJ (日本銀行 為替)、財務省 (JGB)、FRED (米国マクロ)、World Bank Pink Sheet (国際燃料価格)。D-011 で定めた 19 項目メタデータスキーマに従い、各系列に source_name / source_url / license / observation_cutoff / as-of / freshness_sla_days を明示しています。引用元として学術論文・報道記事・行政資料での使用を想定した設計です。

## Q3. なぜ無料で公開できるのか? 収益化モデルは?

A. **収益化はしません**。一般社団法人エネルギー情報センター (代表理事 江田健次) の公益事業として運営し、月額運用費 17,400 円 (GA 後) は EIC 法人で全額負担します。広告・課金・有料プラン・データ販売の予定なし (D-002 + D-012)。北極星「日本のエネルギーと金融の引用インフラ」を**永続的な公共財**として提供する方針です。スポンサーシップやクラウドファンディングも当面は予定していません。媒体・研究機関からの問い合わせは kenji.eda@gmail.com で受け付けます。

## Q4. 朝刊サマリーは AI が書いているのか?

A. `/today` 朝刊サマリーは **テンプレート駆動の自動生成**で、AI による創作ではありません。前日終値・前日比・週次変化率・季節性比較などの統計値を 5 系列横断で計算し、事前に定義した日本語テンプレートに当てはめて毎平日朝 7:00 JST に配信します。GitHub Actions の cron で自動更新、編集者バッジ + トレンドアラート (異常変動の検出) を併記しています。文章の最終的な妥当性は人間 (リン編集軸) が事後レビューする運用です。LLM による創発的な解釈は混入させません。

## Q5. トレーディング会社や金融機関は利用できるのか?

A. はい、利用できます。CC BY 4.0 ライセンスに従い、出典表示 (「出典: EIC Data, accessed YYYY-MM-DD, CC BY 4.0」相当) を付ければ、社内レポート・顧客向け資料・アルゴリズム入力・ストラテジー資料への組込みが可能です。7 つの利用シーン: ①日次マクロモニタリング、②燃料 × 為替の相関分析、③電力卸価格と気象のヘッジ判断、④JGB と米国金利スプレッド分析、⑤エネルギー関連株の前提整理、⑥クライアント向けスポット資料、⑦内部研修・新人教育。データ販売はしません、API 利用も無料です。

## Q6. 学術引用に使えるか? BibTeX 形式は信頼できるか?

A. 学術引用での利用を**最初から想定した設計**です。各 Insight・catalog ページに常設の CitationButton から BibTeX / Chicago 17 / APA 7 の 3 形式をワンクリックでコピーできます。BibTeX エントリには title / author / year / month / publisher / url / urldate / note が含まれ、Chicago Manual of Style 17th edition および APA 7th edition の Data Citation Style に準拠しています。データ系列は Dataset 型、Insight 記事は Article 型で生成、accessed_at (アクセス日付) も自動付与されます。`/citation-policy` ページで早見表を公開しています。

## Q7. GA4 + GSC で個人情報は収集していないか?

A. 直接識別情報 (氏名・メールアドレス・電話番号など) は**一切収集しません**。GA4 は IP 匿名化 (最後のオクテットを 0 に置換、Google 側設定) + Cookie (`_ga` / `_ga_*`) + 訪問ページ URL + 滞在時間 + カスタムイベント 4 種 (csv_download / cite_copy / external_source_click / filter_change) のみを取得します。GSC は検索クエリ + 表示回数 + クリック数のみ。サイト改善目的に限定使用、第三者提供は Google 以外なし。詳細は `/privacy` (改正個人情報保護法 PIPL 2022 + GDPR 整合済、リク監修済) を参照ください。Cookie 無効化でも全機能が利用可能です。

## Q10. Cowork mode + Claude Code を使った理由は?

A. **個人運営の公益事業で 21 日スプリントを実現するため**です。Cowork mode は EDA 代表 + リン (編集軸) + リク (法務監修) + ナギ (デザイン) + アオ (リサーチ) の 5 役割を時間軸で切替える進行モデル、Claude Code は Anthropic 製の CLI 型コーディングエージェントです。役割分担の明示 + 自動化された実装層を組合せることで、人手作業を最小化しつつ、編集判断・法務監修・品質確認は人間が責任を持つ構造を維持しました。当初 55 日想定が 21 日で完走、5/12 に機能完成日達成 (当初 6/14 から 33 日前倒し) + 統合テスト全 3 日完走 (Lighthouse 全 12 主要ページ Perf 平均 0.972 / CLS 平均 0.004 / 全 207 URL HTTP 200)、5/11-12 で 14 連続マイルストーンを樹立できたのは、この体制の成果と評価しています。
